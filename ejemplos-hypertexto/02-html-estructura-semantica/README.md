# HTML Estructura Semántica

**Concepto:** Usar etiquetas HTML semánticas para un código más significativo

## Archivos
- `index.html` - Documento con etiquetas semánticas

## Conceptos Demostrados
- `<header>` - Encabezado principal de la página
- `<nav>` - Navegación (menú)
- `<main>` - Contenido principal
- `<article>` - Contenido independiente y relevante
- `<section>` - Agrupación temática de contenido
- `<aside>` - Contenido secundario (barra lateral)
- `<footer>` - Pie de página
- Importancia de la semántica para SEO y accesibilidad

## Diferencia con Concepto 01
- Usa `<div>` genéricos → Usa `<article>`, `<section>`, `<nav>` específicos
- Mejor para motores de búsqueda
- Mejor para lectores de pantalla (accesibilidad)
- Más fácil de entender para otros desarrolladores

## Diferencias Principales
| Concepto 01 | Concepto 02 |
|---|---|
| Muchos `<div>` genéricos | Etiquetas semánticas específicas |
| No descriptor el propósito | Estructura significativa |
| Menos SEO | Mejor SEO |

## Resultado Esperado
Una página bien estructurada con significado claro en el código HTML.
